Project Title

Deus Ex Securitas

Project Summary

An AI auditing system, accessible to anyone, lighting fast, and scalable for the future

Problem & Status Quo

Smart contract vulnerabilities are a problem that has costed the blockchain industry over $3B, a number that will only grow as the industry expands. It’s hard to defend against exploits and the only solution we currently have is manually audit smart contracts. The role that security audits play is to make sure that contracts don’t contain any technical vulnerabilities and to provide specialized consultation. However, manual security audits come with a set of problems:

  • Cost: Auditing services cost $50k - $150k+ gate-keeping those who don’t have sufficient capital
  • Time: Smart contract audits take weeks or months to complete, slowing iteration and release cycles
  • Inability to scale: Auditing services can’t increase their throughput without sacrificing quality or hiring more auditors
  • Human Errors: Despite their skills, exploits can still fall through the cracks of the best of auditors

What’s the root cause behind all these issues? It’s that auditors have a limited amount of time and they’re wasting a huge portion of that time trying to fix the technical issues in their clients’ code. Time that could have otherwise been spent doing their job and providing a consultation.

Our Solution

So let’s introduce artificial intelligence into the loop. Let’s create a model capable of detecting exploits and solving the technical issues normally found by humans. This means it can become a powerful tool in an auditor’s belt that helps solve each of the problems outlined above.

  • Cost: You can run the model on your computer for free
  • Time: It takes seconds to run the model
  • Scalability: Software can be upgraded instantly, and duplicated infinitely
  • Generalization: Machines are capable of learning from all known exploits and can extrapolate to find novel exploits outside of their training set

We plan on integrating this product into manual auditing firms. ML will deal with the technical problems and auditors will be freed to provide specialized consultation. This helps auditing firms in the following areas:

  • Time: Auditors don’t have to deal with fixing the code thus speeding things up
  • Throughput: A byproduct of speed is increased throughput, thus scalability
  • Cost: A byproduct of speed is ability to decrease the cost per audit - offering more competitive and affordable rates
  • Service: Auditors are freed to do their real job, which is specialized consultation which means more care into the service

Our Vision

We envision a future where Deus Ex Securitas becomes integrated into developer and auditing workflows. For developers, we’re here to be a fast, accessible, accurate, and scalable tool to make all of crypto safer from exploits. For auditing firms we will be the tool to handle all the technical issues allowing them to provide specialized consultation. We hope to become a base security layer for all of crypto - to create a future where exploits cease to exist.

Technical Roadmap

Version 1 - Proof of Intelligence

Capable of detecting any smart contracts that contain the re-entry vulnerability

Technical Requirements

  • Larger dataset
  • Better-trained model
  • Improved tests & validation
  • Improved server-side architecture

Non-Technical Requirements

  • Set up an advisory board
  • Speak with auditing firms & end users

Estimated Time

  • 2-4 weeks

Estimated Resources

  • No extra resources required

Version 2 - Expansion to Other Exploits

Same as V1, but ability to detect N different vulnerabilities (Where N is determined by talking to auditing companies and seeing if they have a checklist)

Technical Requirements

  • Create a unique dataset for each vulnerability
  • Improve the model architecture to improve generalizability

Non-Technical Requirements

  • Add a business person to the advisory board and begin creating an alpha product

Estimated Time

  • 3-4 weeks
  • Limiting Factor
    • Gathering a sufficiently large dataset
    • Training our model from scratch with a new architecture & tokenization system

Estimated Resources

  • ~$10,000 to create dataset bounties to speed up the dataset generation process

Post V2

  • Turn Version 2 into an alpha product
  • Begin partnering with Hackathons to get people to try out our product
  • Implement vulnerability highlighting into the model
  • Higher a ML engineer to help us with Version 3's technical challenges

Version 3 - Few-Shot & Generalization

Ability to recognise an arbitrary number of vulnerabilities without needing to be re-trained based on a small number of examples

Technical Requirements

  • Higher ML engineers to help create novel tech to solve our problems
  • Create a much larger, more variable, dataset with a broader scope
  • Powerful hardware will likely become necessary as our model grows in size

Non-Technical Requirements

  • A greater understanding of security so we can fine-tune the model - we will either need to learn this ourselves or higher an expert

Estimated Time

  • 3-6 months
  • Limiting Factor
    • New technology needs to be developed
    • Training times for models will increase
    • Salaries for employees

Estimated Resources

  • Cloud servers / Dedicated hardware
  • An employee with a focus on security & machine learning

Post V3

  • Create version 1.0 of our product
  • Partner with auditing firms and integrate our product into their toolbox

Version 4 - Superhuman Performance

End Game: create models that are capable to detecting exploits that were never seen before. Surpass human performance.

Technical Requirements

  • ML will likely be the bottleneck again, so smart engineers will be needed
  • Much larger dataset with a broader scope that includes lots of edge cases & complex vulnerabilities
  • Contextual detecting that can detect application-specific vulnerabilities

Non-Technical Requirements

  • A larger team to help build the model

Estimated Time

  • Unknown
  • Limiting Factor
    • To our knowledge this surpasses current ML capabilities - new technology will need to be developed
    • Training costs for very large models
    • Salaries for employees

Estimated Resources

  • Lots of new hardware / Cloud resources
  • New employees to help with ML breakthroughs

Post V4

  • Create version 2.0 of our product
  • Broad sweep of existing smart contracts - we would need to notify people if we find problems in their code that could be easily exploited. Then release our systems to everyone

Progress

The project was initially launched at ETHDenver as a Hackathon project. We’ve had a tremendous amount of positive feedback! We won 3rd place in the community vote and won Vitalik’s choice as his favourite project!
